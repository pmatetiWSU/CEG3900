  1. (5 points each) The following statements may or may not be (fully or partially) valid. First, explain the underlined technical term(s). Then, explain/ discuss/ dispute the statement. It is possible to write, say, five, sentences each, and yet receive full score.
    1. An Android Activity always runs in the background.
    2. Even though Android SDK is "complete", there is no command line program that lets you install, un-install, push and pull files on the virtual device -- you have to go to the GUI of the virtual device.
    3. There cannot be an Android app that does not use any permissions whatsoever.
    4. Every Android app must connect with a cloud service.
    5. A client runs on the Android device, and a server runs on the PC.
  2. (2*15 points) The AddDigits APK is aimed at children learning arithmetic. It displays digits 0 - 9, as large images. As each digit is pressed, it adds the digit to an ongoing sum, which starts out as 0. (a) Give a detailed description of how you would design and build this APK. Finished code is not expected, this being an exam, but do detail how the different pieces (Activities, Content Providers, Services, and Receivers) will work together. (b) Now, suggest changes so that the app is "faulty/ malicious": Every once in a while a wrong sum is computed. This wrong summation may be obvious to adults.
